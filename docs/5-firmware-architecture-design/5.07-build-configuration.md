## 1. Purpose

This view defines everything needed to build ParkSense firmware reproducibly: CMake structure, toolchain configuration, compile-time tunables, per-target source selection, static analysis integration, and the MISRA-C deviation list.

## 2. Build System Overview

| Parameter | Value |
|-----------|-------|
| Build system | CMake ≥ 3.25 |
| Generator | Ninja |
| Cross compiler | `arm-none-eabi-gcc` 13.x |
| Host compiler (tests) | System GCC or Clang (host-native) |
| Docker image | `parksense-toolchain:latest` (deterministic env) |

## 3. Compile-Time Configuration Constants

All tunables are defined as CMake cache variables and injected as preprocessor defines. No magic numbers in source code.

### 3.1 System Timing

| Constant | Default | Unit | Description | CMake Variable |
|----------|---------|------|-------------|----------------|
| `SLEEP_INTERVAL_S` | 30 | seconds | Main cycle sleep period | `-DSLEEP_INTERVAL_S=30` |
| `HEARTBEAT_INTERVAL_S` | 60 | seconds | Heartbeat transmission period | `-DHEARTBEAT_INTERVAL_S=60` |
| `IWDG_TIMEOUT_MS` | 4000 | ms | Independent watchdog timeout | `-DIWDG_TIMEOUT_MS=4000` |
| `IWDG_FEED_INTERVAL_MS` | 2000 | ms | Watchdog feed period | `-DIWDG_FEED_INTERVAL_MS=2000` |

### 3.2 PDM Thresholds

| Constant | Default | Unit | Description | CMake Variable |
|----------|---------|------|-------------|----------------|
| `VEHICLE_PRESENCE_THRESHOLD_MM` | 300 | mm | ToF distance below which a vehicle is present | `-DVEHICLE_PRESENCE_THRESHOLD_MM=300` |
| `MAGNETIC_DISTURBANCE_THRESHOLD_MG` | 200 | mGauss | Magnetometer delta above which a vehicle is present | `-DMAGNETIC_DISTURBANCE_THRESHOLD_MG=200` |
| `TOF_SIGMA_THRESHOLD` | 15 | — | VL53L5CX sigma filter: discard zones with sigma > this | `-DTOF_SIGMA_THRESHOLD=15` |
| `N_ENTER` | 3 | samples | Consecutive triggered samples to enter OCCUPIED | `-DN_ENTER=3` |
| `M_EXIT` | 5 | samples | Consecutive clear samples to exit OCCUPIED | `-DM_EXIT=5` |

### 3.3 CPM / Network

| Constant | Default | Unit | Description | CMake Variable |
|----------|---------|------|-------------|----------------|
| `CPM_QUEUE_DEPTH` | 16 | slots | CPM circular queue depth | `-DCPM_QUEUE_DEPTH=16` |
| `CPM_MAX_RETRIES` | 3 | — | Max TX retries before failure | `-DCPM_MAX_RETRIES=3` |
| `CPM_BACKOFF_BASE_MS` | 100 | ms | First retry backoff period | `-DCPM_BACKOFF_BASE_MS=100` |
| `MAX_JOIN_RETRIES` | 5 | — | Max Zigbee join attempts before standby | `-DMAX_JOIN_RETRIES=5` |
| `JOIN_TIMEOUT_MS` | 30000 | ms | Zigbee scan timeout | `-DJOIN_TIMEOUT_MS=30000` |
| `JOIN_RETRY_DELAY_S` | 60 | seconds | Delay between join retries | `-DJOIN_RETRY_DELAY_S=60` |

### 3.4 Network Transport Selection (Gateway only)

| Define | Description |
|--------|-------------|
| `-DNET_TRANSPORT_WIFI` | Compile `drivers/net/wifi/wifi_driver.c` (EMW3080B via STM32_Network_Library) — **Phase 1 default** |
| `-DNET_TRANSPORT_ETHERNET` | Compile `drivers/net/ethernet/eth_driver.c` (WIZnet W5500 via ioLibrary) — Phase 2 |

Exactly one of these must be passed to the gateway build. The `net_api.h` interface is identical regardless of selection. The node target links neither.

### 3.4 Zigbee RF

| Constant | Default | Description | CMake Variable |
|----------|---------|-------------|----------------|
| `ZIGBEE_CHANNEL` | 15 | IEEE 802.15.4 channel (11–26) | `-DZIGBEE_CHANNEL=15` |
| `ZIGBEE_PAN_ID` | 0x1A2B | PAN identifier | `-DZIGBEE_PAN_ID=0x1A2B` |
| `ZIGBEE_TX_POWER_DBM` | 0 | TX power in dBm | `-DZIGBEE_TX_POWER_DBM=0` |

## 4. CMake Target Structure

### 4.1 Top-Level `CMakeLists.txt`

```cmake
cmake_minimum_required(VERSION 3.25)
project(ParkSense C ASM)

# ── Toolchain ──
set(CMAKE_TOOLCHAIN_FILE cmake/toolchain-arm-none-eabi.cmake)
include(cmake/targets.cmake)
include(cmake/static-analysis.cmake)

# ── Firmware version (injected into ps_version.h) ──
set(FW_VERSION_MAJOR 0 CACHE STRING "Firmware major version")
set(FW_VERSION_MINOR 1 CACHE STRING "Firmware minor version")
set(FW_VERSION_PATCH 0 CACHE STRING "Firmware patch version")

configure_file(common/ps_version.h.in common/ps_version.h @ONLY)

# ── Build targets ──
add_subdirectory(platform)
add_subdirectory(hal)
add_subdirectory(bsp)
add_subdirectory(drivers)
add_subdirectory(pdm)
add_subdirectory(cpm)
add_subdirectory(app)
add_subdirectory(bootloader)

# ── Host-compiled unit tests (only when not cross-compiling) ──
if(NOT CMAKE_CROSSCOMPILING)
    enable_testing()
    add_subdirectory(tests)
endif()
```

### 4.2 Target Definitions (`cmake/targets.cmake`)

```cmake
# ── IoT Node target ──
add_executable(parksense_node.elf)
target_compile_definitions(parksense_node.elf PRIVATE
    TARGET_NODE
    STM32U585xx
    USE_HAL_DRIVER
    SLEEP_INTERVAL_S=${SLEEP_INTERVAL_S}
    HEARTBEAT_INTERVAL_S=${HEARTBEAT_INTERVAL_S}
    VEHICLE_PRESENCE_THRESHOLD_MM=${VEHICLE_PRESENCE_THRESHOLD_MM}
    MAGNETIC_DISTURBANCE_THRESHOLD_MG=${MAGNETIC_DISTURBANCE_THRESHOLD_MG}
    N_ENTER=${N_ENTER}
    M_EXIT=${M_EXIT}
    CPM_QUEUE_DEPTH=${CPM_QUEUE_DEPTH}
    CPM_MAX_RETRIES=${CPM_MAX_RETRIES}
)
target_link_options(parksense_node.elf PRIVATE
    -T${CMAKE_SOURCE_DIR}/platform/stm32u585xx_node.ld
    -Wl,--gc-sections
    -Wl,-Map=parksense_node.map
)

# ── Gateway target ──
add_executable(parksense_gateway.elf)
target_compile_definitions(parksense_gateway.elf PRIVATE
    TARGET_GATEWAY
    STM32U585xx
    USE_HAL_DRIVER
    CPM_QUEUE_DEPTH=${CPM_QUEUE_DEPTH}
    NET_TRANSPORT_WIFI   # Phase 1 default; swap to NET_TRANSPORT_ETHERNET for W5500
)
target_link_options(parksense_gateway.elf PRIVATE
    -T${CMAKE_SOURCE_DIR}/platform/stm32u585xx_gateway.ld
    -Wl,--gc-sections
    -Wl,-Map=parksense_gateway.map
)

# ── Bootloader target ──
add_executable(parksense_boot.elf)
target_compile_definitions(parksense_boot.elf PRIVATE
    STM32U585xx
    USE_HAL_DRIVER
    BOOTLOADER_BUILD
)
target_link_options(parksense_boot.elf PRIVATE
    -T${CMAKE_SOURCE_DIR}/bootloader/stm32u585xx_boot.ld
    -Wl,--gc-sections
)
```

## 5. Toolchain Configuration (`cmake/toolchain-arm-none-eabi.cmake`)

```cmake
set(CMAKE_SYSTEM_NAME Generic)
set(CMAKE_SYSTEM_PROCESSOR arm)

set(CMAKE_C_COMPILER   arm-none-eabi-gcc)
set(CMAKE_ASM_COMPILER arm-none-eabi-gcc)
set(CMAKE_OBJCOPY      arm-none-eabi-objcopy)
set(CMAKE_SIZE         arm-none-eabi-size)

set(CMAKE_C_FLAGS_INIT
    "-mcpu=cortex-m33 -mthumb -mfloat-abi=hard -mfpu=fpv5-sp-d16"
)
set(CMAKE_C_FLAGS
    "${CMAKE_C_FLAGS_INIT} -Wall -Wextra -Werror -std=c11 -ffunction-sections -fdata-sections"
)
set(CMAKE_C_FLAGS_DEBUG   "-Og -g3 -DDEBUG")
set(CMAKE_C_FLAGS_RELEASE "-Os -DNDEBUG -flto")

set(CMAKE_EXE_LINKER_FLAGS_INIT
    "-mcpu=cortex-m33 -mthumb -mfloat-abi=hard -mfpu=fpv5-sp-d16 --specs=nosys.specs --specs=nano.specs"
)
```

**Key flags:**
- `-Werror`: All warnings are treated as errors — no suppressions without documented deviation
- `-ffunction-sections -fdata-sections` + `-Wl,--gc-sections`: Dead code elimination
- `--specs=nano.specs`: Newlib-nano for smaller libc
- `--specs=nosys.specs`: No OS syscalls (bare metal)
- `-Os` for release: Optimize for size (flash budget)
- `-flto` for release: Link-time optimization

## 6. Compiler Warning Policy

| Flag | Status |
|------|--------|
| `-Wall` | Enabled |
| `-Wextra` | Enabled |
| `-Werror` | Enabled |
| `-Wpedantic` | Enabled in CI, disabled locally (vendor headers generate pedantic warnings) |
| `-Wconversion` | Enabled |
| `-Wshadow` | Enabled |
| `-Wdouble-promotion` | Enabled (critical — Cortex-M33 FPU is single-precision only) |
| `-Wformat=2` | Enabled |
| `-Wundef` | Enabled |
| `-Wno-unused-parameter` | Enabled for vendor source directories only |

## 7. Static Analysis (`cmake/static-analysis.cmake`)

### 7.1 cppcheck

```cmake
find_program(CPPCHECK cppcheck)
if(CPPCHECK)
    set(CMAKE_C_CPPCHECK
        ${CPPCHECK}
        --enable=all
        --suppress=missingIncludeSystem
        --suppress=unusedFunction
        --inline-suppr
        --error-exitcode=1
```

## 8. Version Injection

The firmware version is injected at build time from CMake variables, Git tags, or CI/CD pipeline:

**Template (`common/ps_version.h.in`):**

```c
#ifndef PS_VERSION_H
#define PS_VERSION_H

#define FW_VERSION_MAJOR  @FW_VERSION_MAJOR@
#define FW_VERSION_MINOR  @FW_VERSION_MINOR@
#define FW_VERSION_PATCH  @FW_VERSION_PATCH@

#define FW_VERSION_U32    ((FW_VERSION_MAJOR << 24) | \
                           (FW_VERSION_MINOR << 16) | \
                           (FW_VERSION_PATCH << 8))

#define FW_VERSION_STRING "@FW_VERSION_MAJOR@.@FW_VERSION_MINOR@.@FW_VERSION_PATCH@"

#endif
```

## 9. Post-Build Steps

After linking each `.elf` target:

```cmake
# Generate raw binary for flashing
add_custom_command(TARGET parksense_node.elf POST_BUILD
    COMMAND ${CMAKE_OBJCOPY} -O binary parksense_node.elf parksense_node.bin
    COMMAND ${CMAKE_SIZE} parksense_node.elf
    COMMENT "Generating binary and size report"
)

# Generate Intel HEX for STM32CubeProgrammer
add_custom_command(TARGET parksense_node.elf POST_BUILD
    COMMAND ${CMAKE_OBJCOPY} -O ihex parksense_node.elf parksense_node.hex
)
```

## 10. Image Signing (CI/CD Step)

The binary is signed after build, before artifact publication:

```bash
# Run in CI/CD (GitHub Actions)
python3 tools/sign_image.py \
    --binary build/parksense_node.bin \
    --key-file /run/secrets/ecdsa_private_key.pem \
    --version ${FW_VERSION_MAJOR}.${FW_VERSION_MINOR}.${FW_VERSION_PATCH} \
    --output build/parksense_node_signed.bin
```

This prepends the 256-byte `ps_image_header_t` (MAGIC, VERSION, CRC-32, SHA-256, ECDSA P-256 signature) to the raw binary. See [[4.6-security-architecture]] §4.2.

## 12. MISRA-C:2012 Deviation List

ParkSense targets MISRA-C:2012 compliance for all project-written code (not vendor source). The following deviations are documented and justified:

| Rule | Category | Description | Deviation Justification |
|------|----------|-------------|------------------------|
| Rule 11.3 | Required | Cast between pointer to object type and pointer to different object type | Used in `APP_FaultLog()` for `uint64_t` flash programming from `fault_record_t`. Controlled, aligned, size-verified. |
| Rule 11.4 | Advisory | Cast from pointer to integer or vice versa | Used in bootloader for reading flash at absolute addresses (linker symbols). Unavoidable in memory-mapped embedded. |
| Rule 14.3 | Required | Controlling expression shall not be invariant | `while(1)` in super loop is intentional. |
| Rule 17.7 | Required | Return value of non-void function shall be used | `HAL_FLASH_Lock()` returns `HAL_StatusTypeDef`, always `HAL_OK` when correctly sequenced. Return value discarded intentionally. |
| Rule 20.10 | Advisory | `#` and `##` preprocessor operators | Used in `PS_STATIC_ASSERT` macro. Standard C11 `_Static_assert` is preferred where possible. |
| Rule 21.6 | Required | `<stdio.h>` functions shall not be used | `printf` used only in `DEBUG` builds via UART for development. Compiled out in release via `#ifdef DEBUG`. |

**Vendor source exclusion:** Files in `vl53l5cx_uld/`, `iis2mdctr_reg.c`, `mx_wifi/`, and `hal/` are excluded from MISRA analysis (third-party, unmodified). Only ParkSense-authored code is subject to the deviation list.

## 14. Docker Build Environment

```dockerfile
FROM ubuntu:22.04
RUN apt-get update && apt-get install -y \
    cmake ninja-build \
    gcc-arm-none-eabi \
    python3 python3-pip \
    cppcheck \
    git
RUN pip3 install ecdsa pyasn1  # for sign_image.py
WORKDIR /firmware
```

**Usage:**

```bash
docker build -t parksense-toolchain .
docker run --rm -v $(pwd):/firmware parksense-toolchain \
    cmake -B build -G Ninja --preset release-node && \
    cmake --build build
```

Identical environment for local development and GitHub Actions CI.
