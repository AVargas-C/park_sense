## 1. Purpose

This view defines every significant data type used in ParkSense firmware. Each type is documented with its layout, size, alignment, and the module that owns it. This serves as the single source of truth for developers implementing or reviewing the firmware.

---

## 2. No-Heap Policy (FD-001)

ParkSense firmware does not use dynamic memory allocation.

| Constraint | Enforcement |
|------------|-------------|
| `malloc`, `calloc`, `realloc`, `free` | Not linked — symbol undefined at link time |
| Heap section in linker script | `_Min_Heap_Size = 0` |
| MISRA-C Rule 21.3 | Violation detected by cppcheck + PC-lint |

**Rationale:**
- Deterministic memory footprint — the firmware uses exactly the same amount of RAM on every execution
- No fragmentation — critical for 5+ year unattended operation
- No use-after-free, double-free, or dangling pointer risks
- Stack + static allocation is sufficient for all ParkSense data

**Alternative:** All buffers are statically allocated with known compile-time sizes. Circular queues use static arrays with head/tail indices.

## 6. CPM Types (`cpm/cpm_packet.h`)

### 6.1 Packet Structure

```c
/* Message type constants */
#define CPM_MSG_OCCUPANCY   0x01
#define CPM_MSG_HEARTBEAT   0x02
#define CPM_MSG_ACK         0x03

/* Occupancy field values */
#define CPM_OCC_FREE        0x00
#define CPM_OCC_OCCUPIED    0x01
#define CPM_OCC_ERROR       0xFF

/* Fixed packet size */
#define CPM_PACKET_SIZE     12

/* CPM packet — wire format (little-endian) */
typedef struct __attribute__((packed)) {
    uint8_t  msg_type;      /* offset 0:  CPM_MSG_* */
    uint16_t node_id;       /* offset 1:  16-bit device ID */
    uint16_t space_id;      /* offset 3:  parking space ID */
    uint8_t  occupancy;     /* offset 5:  CPM_OCC_* */
    uint32_t timestamp;     /* offset 6:  epoch seconds (gateway-stamped) */
    uint16_t crc16;         /* offset 10: CRC-16/CCITT-FALSE over [0:9] */
} cpm_packet_t;

PS_STATIC_ASSERT(sizeof(cpm_packet_t) == CPM_PACKET_SIZE,
                 "cpm_packet_t must be exactly 12 bytes");
```

### 6.2 Serialization Buffer

```c
/* Serialized byte buffer for RF transmission */
typedef struct {
    uint8_t  raw[CPM_PACKET_SIZE];
    uint16_t len;       /* always CPM_PACKET_SIZE after serialization */
} cpm_buf_t;
```

## 7. Circular Queue (`cpm/cpm_queue.h`)

```c
#define CPM_QUEUE_DEPTH  16

typedef struct {
    cpm_packet_t slots[CPM_QUEUE_DEPTH];
    uint8_t      head;          /* next write index */
    uint8_t      tail;          /* next read index */
    uint8_t      count;         /* items in queue */
} cpm_queue_t;
```

**Properties:**
- Fixed-size, statically allocated — no `malloc`
- Single producer / single consumer — lock-free in ISR-to-main scenario (but ParkSense uses main context only for both push and pop)
- Overflow policy: drop oldest (overwrite tail — newest event always preserved; latest state is ground truth)

**API:**

```c
void       CPM_Queue_Init(cpm_queue_t *q);
ps_status_t CPM_Queue_Push(cpm_queue_t *q, const cpm_packet_t *pkt);  /* PS_ERR_FULL on overflow */
ps_status_t CPM_Queue_Pop(cpm_queue_t *q, cpm_packet_t *pkt);         /* PS_ERR_EMPTY if empty */
bool       CPM_Queue_IsEmpty(const cpm_queue_t *q);
bool       CPM_Queue_IsFull(const cpm_queue_t *q);
```

## 8. Retransmission Context (`cpm/cpm_retry.h`)

```c
#define CPM_MAX_RETRIES     3
#define CPM_BACKOFF_BASE_MS 100     /* 100 ms, 200 ms, 400 ms */

typedef struct {
    cpm_packet_t pending_pkt;       /* packet awaiting ACK */
    uint8_t      retry_count;       /* 0..CPM_MAX_RETRIES */
    uint32_t     next_retry_tick;   /* HAL_GetTick() deadline for next retry */
    bool         awaiting_ack;      /* true if TX in flight */
} cpm_tx_ctx_t;
```

**Backoff calculation:**

```c
uint32_t backoff_ms = CPM_BACKOFF_BASE_MS << retry_count;
/* retry 0: 100 ms, retry 1: 200 ms, retry 2: 400 ms */
```

## 13. ISR-to-Main Volatile Flags

All ISR-to-main communication is via dedicated volatile flags — no shared data structures, no mutexes.

```c
/* Declared in bsp.c, extern'd in bsp.h */
volatile bool g_rtc_alarm_flag;         /* set by RTC_Alarm_IRQHandler */
volatile bool g_wifi_data_ready;        /* set by EXTI_WiFi_IRQHandler (gateway) */
volatile bool g_zigbee_rx_pending;      /* set by IPCC_C1_RX_IRQHandler */
```

**Access pattern:**

```c
/* Main loop (consumer) */
if (g_rtc_alarm_flag) {
    g_rtc_alarm_flag = false;   /* clear — single consumer, no race */
    /* process wake */
}
```

**Safety:** Single producer (ISR) sets flag to `true`. Single consumer (main loop) reads and clears. Both operations are atomic on Cortex-M33 for `bool` (1-byte aligned). The `volatile` qualifier prevents compiler reordering.

## 14. Data Structure Size Summary

| Type | Size | Count | Total SRAM |
|------|------|-------|------------|
| `pdm_fsm_ctx_t` | 8 B | 1 | 8 B (SRAM2) |
| `pdm_calibration_t` | 12 B | 1 | 12 B (SRAM2) |
| `cpm_packet_t` | 12 B | — | — (transient) |
| `cpm_queue_t` (TX) | 12×16 + 3 = 195 B | 1 | 195 B (SRAM1) |
| `cpm_queue_t` (RX) | 195 B | 1 | 195 B (SRAM1) |
| `cpm_tx_ctx_t` | 20 B | 1 | 20 B (SRAM1) |
| `rf_frame_t` | 136 B | 1 | 136 B (SRAM1) |
| `VL53L5CX_ResultsData` | 292 B | 2 (double buf) | 584 B (SRAM1) |
| `sched_ctx_t` | ~136 B | 1 | 136 B (SRAM1) |
| `fault_record_t` | 64 B | 1 (staging) | 64 B (SRAM1) |
| Volatile flags | 1 B each | 3 | 3 B (SRAM1) |
| **Total static allocation** | | | **~1,353 B** |

> Total static SRAM usage is approximately 1.4 KB — well within the 192 KB SRAM1 + 64 KB SRAM2 budget.
