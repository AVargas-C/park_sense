## 1. Purpose

This document defines the binary layout and C type definitions for the three persistent metadata structures in ParkSense firmware:

1. **`ps_image_header_t`** — 256-byte header prepended to every application image. Consumed by the bootloader to validate authenticity, integrity, target compatibility, and anti-rollback before branching to the application.
2. **`ota_control_t`** — 16-byte structure in the OTA Control flash sector (`0x081E2000`). Directs the bootloader to attempt an OTA slot swap and enforces the anti-rollback counter floor.
3. **`secure_config_t`** — Structure in the Secure Config flash sector (`0x081E0000`). Holds the ECDSA public key, per-device Install Code, Zigbee NWK key, and the absolute anti-rollback floor. Only the bootloader and Secure-world veneer functions access this struct directly.

All three structures live in flash. All offsets are fixed — changing them requires a coordinated update of the signing tool, bootloader, and linker scripts.

## 2. Application Image Header — `ps_image_header_t`

### 2.1 Location

The header occupies the first 256 bytes (`0x00–0xFF`) of each application slot:

| Slot | Header address | First executable instruction |
|------|---------------|------------------------------|
| App A | `0x08020000` | `0x08020100` |
| App B | `0x08100000` | `0x08100100` |

The NS vector table is placed by the linker at `slot_base + 0x100`. `VTOR_NS` is set by the bootloader to `slot_base + 0x100` after validation.

### 2.2 Field Map

```
Offset  Size  Type        Field                    Description
──────  ────  ──────────  ───────────────────────  ─────────────────────────────────────────────
0x00     4    uint32_t    magic                    0x504B5346  ("PKSF" in ASCII)
0x04     1    uint8_t     major                    Semantic version — major component
0x05     1    uint8_t     minor                    Semantic version — minor component
0x06     2    uint16_t    patch                    Semantic version — patch component
0x08     4    uint32_t    rollback_counter         Monotonically increasing; bootloader checks ≥ min
0x0C     4    uint32_t    image_size               Byte count of image body (after this header)
0x10     4    uint32_t    crc32                    CRC-32/MPEG-2 over image body only
0x14     4    uint32_t    build_timestamp          Unix timestamp (seconds) at build time
0x18     1    uint8_t     target_id                0x01 = Node, 0x02 = Gateway
0x19     1    uint8_t     hwrev_min                Minimum PCB hardware revision this image requires
0x1A     2    uint16_t    signing_key_id           Key pair identifier (for key rotation)
0x1C     4    uint32_t    min_btl_version_packed   Minimum bootloader version required (maj<<16|min<<8|pat)
0x20    64    uint8_t[64] signature                ECDSA P-256 signature: r(32 B) ‖ s(32 B)
0x60     4    uint32_t    build_flags              Compile-time feature bitmask (see §2.4)
0x64     4    uint32_t    header_version           Header format version = 0x00000001
0x68     8    uint8_t[8]  git_commit               Abbreviated git commit SHA (first 8 bytes)
0x70    32    uint8_t[32] sha256_digest            SHA-256 of (header[0x00–0x1F] ‖ image body)
0x90   112    uint8_t[112] _padding                Zeroed; reserved for future fields
──────  ────
0x100          Total: 256 bytes
```

### 2.3 C Type Definition

```c
#ifndef PS_IMAGE_HEADER_H
#define PS_IMAGE_HEADER_H

#include <stdint.h>

/* -- Constants ------------------------------------------------------------ */

#define PS_IMAGE_MAGIC           (0x504B5346UL)  /* "PKSF"       */
#define PS_HEADER_VERSION        (0x00000001UL)  /* Format v1    */
#define PS_HEADER_SIZE           (256U)          /* Bytes        */
#define PS_SIGNATURE_OFFSET      (0x20U)         /* Fixed offset */
#define PS_SHA256_DIGEST_OFFSET  (0x70U)         /* Fixed offset */

/* target_id values */
#define PS_TARGET_NODE           (0x01U)
#define PS_TARGET_GATEWAY        (0x02U)

/* Rollback counter absent / invalid sentinel */
#define PS_ROLLBACK_COUNTER_INIT (0x00000001UL)  /* First valid counter value */

/* build_flags bitmask (see §2.4) */
#define PS_BUILD_FLAG_WIFI       (0x00000001UL)
#define PS_BUILD_FLAG_ETHERNET   (0x00000002UL)
#define PS_BUILD_FLAG_DEBUG      (0x00000004UL)
#define PS_BUILD_FLAG_MISRA      (0x00000008UL)

/* -- Struct --------------------------------------------------------------- */

typedef struct
{
    /* --- Identity block: 0x00–0x1F (32 bytes, covered by signature hash) --- */
    uint32_t magic;                  /* 0x00  PS_IMAGE_MAGIC = 0x504B5346        */
    uint8_t  major;                  /* 0x04  Semantic version major             */
    uint8_t  minor;                  /* 0x05  Semantic version minor             */
    uint16_t patch;                  /* 0x06  Semantic version patch             */
    uint32_t rollback_counter;       /* 0x08  Anti-rollback monotonic counter    */
    uint32_t image_size;             /* 0x0C  Image body size in bytes           */
    uint32_t crc32;                  /* 0x10  CRC-32/MPEG-2 over image body      */
    uint32_t build_timestamp;        /* 0x14  Unix seconds at build time         */
    uint8_t  target_id;              /* 0x18  PS_TARGET_NODE / PS_TARGET_GATEWAY */
    uint8_t  hwrev_min;              /* 0x19  Minimum PCB hardware revision      */
    uint16_t signing_key_id;         /* 0x1A  Key pair ID for rotation support   */
    uint32_t min_btl_version_packed; /* 0x1C  Minimum bootloader version needed  */

    /* --- Signature: 0x20–0x5F (64 bytes) --- */
    uint8_t  signature[64U];         /* 0x20  ECDSA P-256: r[32] ‖ s[32]        */

    /* --- Extended metadata: 0x60–0x8F --- */
    uint32_t build_flags;            /* 0x60  Compile-time feature bitmask       */
    uint32_t header_version;         /* 0x64  PS_HEADER_VERSION = 0x00000001     */
    uint8_t  git_commit[8U];         /* 0x68  git rev-parse --short HEAD (8 B)   */
    uint8_t  sha256_digest[32U];     /* 0x70  SHA-256(identity_block ‖ body)     */

    /* --- Padding: 0x90–0xFF (112 bytes) --- */
    uint8_t  _padding[112U];         /* 0x90  Zeroed; reserved for future use    */
} ps_image_header_t;

/* Compile-time size assertion (C11) */
_Static_assert(sizeof(ps_image_header_t) == PS_HEADER_SIZE,
               "ps_image_header_t must be exactly 256 bytes");

#endif /* PS_IMAGE_HEADER_H */
```

### 2.4 `build_flags` Bitmask

| Bit | Mask | Meaning when set |
|-----|------|-----------------|
| 0 | `0x00000001` | `NET_TRANSPORT_WIFI` — built with EMW3080B WiFi driver |
| 1 | `0x00000002` | `NET_TRANSPORT_ETHERNET` — built with WIZnet W5500 driver |
| 2 | `0x00000004` | Debug build (`-Og -g3`); not for production |
| 3 | `0x00000008` | MISRA strict mode enabled in static analysis |
| 4–31 | — | Reserved, must be zero |

Bits 0 and 1 are mutually exclusive. A bootloader that reads `build_flags` can warn (via fault log) if both are set — indicating a misconfigured build.

### 2.5 Hash and Signing Coverage

**What is hashed (input to SHA-256 and then ECDSA):**

```
Input = header[0x00..0x1F]          (identity block, 32 bytes)
      ‖ image_body[0..image_size-1] (application code + data)
```

**Before hashing, the signer and verifier both zero these fields:**

| Field | Offset | Why zeroed before hash |
|-------|--------|----------------------|
| `signature[64]` | 0x20 | Signature cannot cover itself |
| `sha256_digest[32]` | 0x70 | Digest cannot cover itself |
| `_padding[112]` | 0x90 | Must be canonical zero |

The `sha256_digest` field stores the pre-computed digest for fast integrity checks at runtime. An application can verify its own image integrity (corruption detection) by recomputing SHA-256 and comparing to `sha256_digest` — without invoking the PKA. Full ECDSA verification is only done by the bootloader.

### 2.6 Validation Order (Bootloader Steps [4a]–[4f])

| Step | Field(s) checked | Cost |
|------|-----------------|------|
| [4a] | `magic == PS_IMAGE_MAGIC` | < 1 ms |
| [4b] | `target_id` matches compile-time `TARGET_ID` | < 1 ms |
| [4c] | `rollback_counter >= ota_ctrl.min_rollback_counter` | < 1 ms |
| [4d] check | `min_btl_version_packed` ≤ bootloader's own version | < 1 ms |
| [4e] | SHA-256 over identity block + body; compare to `sha256_digest` | ~10 ms |
| [4f] | ECDSA P-256 verify `signature` using key `signing_key_id` | ~12 ms |
| [4g] | CRC-32 over image body; compare to `crc32` | ~5 ms |

Step [4d] (bootloader compatibility) is a fast check added by the `min_btl_version_packed` field: if the image requires a newer bootloader than what is installed, it is rejected before committing any flash writes.

### 2.7 Key Rotation Support

`signing_key_id` is a 16-bit integer. The bootloader maintains a small static table mapping `signing_key_id` → 64-byte public key. Initially there is one entry (key ID `0x0001`). When an ECDSA key pair is rotated:

1. A new key pair is generated offline.
2. A new bootloader image embedding both the old and new public keys is prepared (with old key still valid for the transition window).
3. The first OTA image signed with the new key carries `signing_key_id = 0x0002`.
4. After all devices have updated, the old key entry is removed in a subsequent bootloader-equivalent flash reprovision.

Because the bootloader is immutable in the field, key rotation requires a planned reprovision cycle (RDP regression → chip erase → re-flash). This is an intentional constraint: key rotation is an infrequent, high-ceremony event.

## 3. OTA Control Sector — `ota_control_t`

### 3.1 Location

`0x081E2000` — Bank 2, page 113, 8 KB, Non-Secure. Written by `ota.c` (application layer, Gateway only). Read and conditionally updated by the bootloader.

### 3.2 Field Map

```
Offset  Size  Type      Field                    Description
──────  ────  ────────  ───────────────────────  ─────────────────────────────────────────
0x00     4    uint32_t  swap_magic               OTA_SWAP_MAGIC if swap pending, else 0x00
0x04     4    uint32_t  committed                0x00000001 after successful App B commit
0x08     4    uint32_t  min_rollback_counter     Anti-rollback floor; bootloader enforces ≥
0x0C     4    uint32_t  crc32_ctrl               CRC-32 over this struct bytes [0x00–0x0B]
0x10  8176    uint8_t[] _padding                 Zeroed; fills 8 KB page
```

### 3.3 C Type Definition

```c
#define OTA_SWAP_MAGIC          (0x5AA5AA5AUL)
#define OTA_CTRL_BASE           (0x081E2000UL)
#define OTA_CTRL_HDR_CRC_LEN   (12U)   /* Bytes covered by crc32_ctrl */

typedef struct
{
    uint32_t swap_magic;            /* OTA_SWAP_MAGIC or 0x00000000          */
    uint32_t committed;             /* 0x00000001 after successful commit     */
    uint32_t min_rollback_counter;  /* Minimum rollback_counter to accept     */
    uint32_t crc32_ctrl;            /* CRC-32/MPEG-2 over bytes [0x00–0x0B]  */
    uint8_t  _padding[8176U];       /* Zeroed                                 */
} ota_control_t;

_Static_assert(sizeof(ota_control_t) == 8192U,
               "ota_control_t must fill exactly one 8 KB flash page");
```

### 3.4 Bootloader Interaction

| Bootloader step | Action on `ota_control_t` |
|----------------|--------------------------|
| Step [3] read | Read `swap_magic` + `committed`; recompute CRC over [0x00–0x0B]; compare to `crc32_ctrl`. Mismatch → treat as no swap pending. |
| Step [4c] anti-rollback | Reject if `header.rollback_counter < ota_ctrl.min_rollback_counter` |
| Step [5] commit | Erase page → write: `swap_magic=0`, `committed=1`, `min_rollback_counter=header.rollback_counter`, recomputed `crc32_ctrl` |
| App B fail → fallback | Clear `swap_magic` only; leave `min_rollback_counter` unchanged |

## 4. Secure Configuration Sector — `secure_config_t`

### 4.1 Location

`0x081E0000` — Bank 2, page 112, 8 KB, **Secure** (SECWM2). Written only via the `SEC_Flash_WriteConfig()` veneer function (Secure world). Never accessible from NS code. Provisioned at manufacturing time.

### 4.2 Field Map

```
Offset  Size  Type        Field                      Description
──────  ────  ──────────  ─────────────────────────  ──────────────────────────────────────────
0x00     4    uint32_t    magic                      0x53434647 ("SCFG")
0x04     4    uint32_t    crc32_cfg                  CRC-32 over struct bytes [0x08..end]
0x08    64    uint8_t[64] ecdsa_public_key           ECDSA P-256 uncompressed public key Qx‖Qy
0x48     4    uint32_t    key_table_version          Version of the key table layout
0x4C     2    uint16_t    active_signing_key_id      Currently active signing key ID
0x4E     2    uint16_t    _reserved0                 Zeroed
0x50     4    uint32_t    abs_min_rollback_counter   Absolute floor; bootloader checks both this
                                                     and ota_ctrl.min_rollback_counter
0x54    16    uint8_t[16] install_code               Per-device random Install Code (Zigbee)
0x64    16    uint8_t[16] nwk_key                    Zigbee NWK Key (AES-128); runtime only
0x74    64    uint8_t[64] wifi_psk                   WiFi passphrase (gateway only); UTF-8
0xB4    16    uint8_t[16] gateway_token              Pre-shared token for server auth (gateway)
0xC4  3900    uint8_t[]   _padding                   Zeroed; fills remaining 8 KB
```

### 4.3 C Type Definition

```c
#define SCFG_MAGIC              (0x53434647UL)   /* "SCFG" */
#define SCFG_BASE               (0x081E0000UL)

typedef struct
{
    uint32_t magic;                     /* 0x00  SCFG_MAGIC                        */
    uint32_t crc32_cfg;                 /* 0x04  CRC-32 over bytes [0x08..sizeof-1] */
    uint8_t  ecdsa_public_key[64U];     /* 0x08  ECDSA P-256 Qx(32 B) ‖ Qy(32 B)  */
    uint32_t key_table_version;         /* 0x48  Key table layout version           */
    uint16_t active_signing_key_id;     /* 0x4C  Currently trusted signing key ID   */
    uint16_t _reserved0;               /* 0x4E  Zeroed                             */
    uint32_t abs_min_rollback_counter;  /* 0x50  Absolute anti-rollback floor       */
    uint8_t  install_code[16U];         /* 0x54  Zigbee Install Code (TRNG, unique) */
    uint8_t  nwk_key[16U];             /* 0x64  Zigbee NWK Key (AES-128)           */
    uint8_t  wifi_psk[64U];            /* 0x74  WiFi passphrase (gateway, UTF-8)   */
    uint8_t  gateway_token[16U];        /* 0xB4  Server pre-shared auth token       */
    uint8_t  _padding[3900U];          /* 0xC4  Zeroed; fills page to 8 KB          */
} secure_config_t;

_Static_assert(sizeof(secure_config_t) == 8192U,
               "secure_config_t must fill exactly one 8 KB flash page");
```

### 4.4 Bootloader Access

The bootloader reads `secure_config_t` directly from Secure flash (`0x081E0000`) in Secure world — no veneer required. It reads only:
- `magic` + `crc32_cfg` — struct integrity check
- `ecdsa_public_key[64]` + `active_signing_key_id` — ECDSA verification
- `abs_min_rollback_counter` — absolute anti-rollback floor

All other fields (NWK key, WiFi PSK, gateway token) are read by Secure veneer functions at application runtime.

### 4.5 CRC Coverage

`crc32_cfg` covers bytes `[0x08 .. sizeof(secure_config_t) - 1]` — everything after the magic and CRC fields themselves. This allows the bootloader to verify struct integrity without knowing the internal field layout ahead of time.

## 5. Image Signing Toolchain

### 5.1 Build Pipeline Integration

The signing tools in `tools/` run inside Docker as part of the CI/CD release stage. They produce the final `_signed.bin` artifact.

```
arm-none-eabi-objcopy → parksense_<target>.bin   (raw binary, no header)
         │
         ▼
tools/inject_header.py
         │  Reads: git rev, build timestamp, version from git tag, target ID,
         │          image size, hwrev_min and build_flags from CMake cache
         │  Computes: CRC-32/MPEG-2 over image body
         │  Zeroes: signature[64], sha256_digest[32], _padding[112]
         │  Prepends: 256-byte ps_image_header_t (all fields except sig + digest)
         │
         ▼
tools/sign_image.py
         │  Computes: SHA-256 over header[0x00–0x1F] ‖ image_body
         │  Writes: sha256_digest[32] into header at offset 0x70
         │  Signs:  SHA-256 digest using ECDSA P-256 private key
         │            (from FIRMWARE_SIGNING_KEY env var / GitHub Actions Secret)
         │  Writes: signature[64] into header at offset 0x20
         │
         ▼
parksense_<target>_v<M.m.p>_signed.bin   (256-byte header + image body)
```

### 5.2 `inject_header.py` Inputs

| Input | Source |
|-------|--------|
| `magic` | Hardcoded `0x504B5346` |
| `major`, `minor`, `patch` | Git tag (`git describe --tags`) |
| `rollback_counter` | `ROLLBACK_COUNTER` CMake cache variable (manually bumped, never auto-decremented) |
| `image_size` | `os.path.getsize(binary)` |
| `crc32` | Computed over binary bytes |
| `build_timestamp` | `int(time.time())` at signing time |
| `target_id` | `--target node` / `--target gateway` CLI arg |
| `hwrev_min` | `HWREV_MIN` CMake cache variable |
| `signing_key_id` | `SIGNING_KEY_ID` CMake cache variable |
| `min_btl_version_packed` | `MIN_BTL_VERSION` CMake cache variable |
| `build_flags` | Computed from CMake `-D` flags present in build |
| `header_version` | Hardcoded `0x00000001` |
| `git_commit[8]` | `git rev-parse HEAD` (first 8 bytes of raw SHA-1) |

### 5.3 Rollback Counter Policy

The `rollback_counter` is **not** auto-derived from the semantic version. It is a separate monotonically increasing integer stored in `CMakeCache.txt` as `ROLLBACK_COUNTER`. Rules:

- Increment `ROLLBACK_COUNTER` when releasing a security fix or breaking change.
- Never decrement it, even when releasing a patch on an older branch.
- The CI/CD pipeline enforces `ROLLBACK_COUNTER >= ota_ctrl.min_rollback_counter` as a pre-release gate (not just the bootloader).

## 6. Header Injection and Verification at Runtime

### 6.1 Application Self-Check

The running application can verify its own image integrity (without ECDSA) by:

```c
#include "ps_image_header.h"

ps_status_t app_verify_own_image(void)
{
    /* Header is at the start of our flash slot */
    const ps_image_header_t *p_hdr =
        (const ps_image_header_t *)BSP_APP_A_BASE;  /* or APP_B if booted from B */

    /* Recompute SHA-256 over identity block + body */
    uint8_t digest[32U];
    /* ... HASH peripheral call ... */

    if (0 != memcmp(digest, p_hdr->sha256_digest, 32U))
    {
        return PS_ERR_CRC;   /* Image corrupted since boot */
    }
    return PS_OK;
}
```

This is useful for a periodic in-operation integrity check (e.g., every 24 hours, or after a power anomaly) without waking the bootloader.
