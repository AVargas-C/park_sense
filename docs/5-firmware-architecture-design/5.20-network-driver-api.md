## 1. Purpose

This document defines the **Layer 3c — Network Driver API** for the gateway target. It specifies:

- The `net_api.h` abstract interface that CPM (Layer 4) calls exclusively
- Two compile-time implementations: WiFi (EMW3080B) and Ethernet (WIZnet W5500)
- The compile-time transport selection mechanism
- Integration notes for each underlying library

The main design principle: **CPM never calls WiFi or Ethernet functions directly.** It calls only the four functions in `net_api.h`. The transport implementation is invisible to all layers above.

---

## 2. Transport Selection

Exactly one transport is compiled per gateway build, controlled by a preprocessor define passed via CMake:

| CMake flag | Transport compiled | Phase |
|---|---|---|
| `-DNET_TRANSPORT_WIFI` | `drivers/net/wifi/wifi_driver.c` via STM32_Network_Library | Phase 1 — dev kit default |
| `-DNET_TRANSPORT_ETHERNET` | `drivers/net/ethernet/eth_driver.c` via WIZnet ioLibrary | Phase 2 — W5500 breakout |

The node target links neither. The gateway `CMakeLists.txt` enforces that exactly one flag is set (CMake fatal error if both or neither are defined).

## 3. `net_api.h` — Abstract Interface

```c
/* drivers/net/net_api.h — Layer 3c public interface */
/* CPM calls these four functions only. Never calls WiFi or Ethernet APIs directly. */

#ifndef NET_API_H
#define NET_API_H

#include <stdint.h>
#include <stddef.h>

/* ── Return codes ── */
typedef enum {
    NET_OK               = 0,
    NET_ERR_CONNECT      = -1,   /* Transport-level connect failure */
    NET_ERR_SEND         = -2,   /* Send failed or partial */
    NET_ERR_RECV         = -3,   /* Receive timeout or error */
    NET_ERR_TLS          = -4,   /* TLS handshake or certificate error */
    NET_ERR_TIMEOUT      = -5,   /* Operation timed out */
    NET_ERR_NOT_READY    = -6,   /* Module not initialized */
} net_err_t;

/* ── Initialise the network module and underlying transport ── */
/* Must be called once from BSP_Init() or gateway app init sequence.    */
net_err_t NET_Init(void);

/* ── Establish a TCP/TLS connection to the server ── */
/* host: null-terminated hostname or IP string                          */
/* port: TCP port number                                                */
/* Returns NET_OK on success; retries are the caller's responsibility.  */
net_err_t NET_Connect(const char *host, uint16_t port);

/* ── Send data over the established connection ── */
/* buf: pointer to data; len: byte count                                */
/* Returns NET_OK if all bytes sent; NET_ERR_SEND otherwise.            */
net_err_t NET_Send(const uint8_t *buf, size_t len);

/* ── Receive data from the established connection ── */
/* buf: receive buffer; len: buffer size; received: bytes actually read */
/* Blocks until data available or timeout_ms elapses.                   */
net_err_t NET_Recv(uint8_t *buf, size_t len, size_t *received, uint32_t timeout_ms);

/* ── Close the connection and release resources ── */
net_err_t NET_Disconnect(void);

#endif /* NET_API_H */
```

## 4. Implementation A — WiFi (EMW3080B)

### 4.1 Overview

| Property | Value |
|---|---|
| File | `drivers/net/wifi/wifi_driver.c` |
| Compiled when | `-DNET_TRANSPORT_WIFI` |
| Underlying library | `STM32_Network_Library` (part of STM32CubeU5 submodule) |
| Hardware | EMW3080B — pre-installed on B-U585I-IOT02A, SPI2 |
| TCP/IP stack | Onboard the EMW3080B module (ARM Cortex-M4 inside module) |
| TLS | mbedTLS on module — HTTPS without host MCU crypto burden |

### 4.2 Mapping to `net_api.h`

| `net_api.h` call | EMW3080B implementation |
|---|---|
| `NET_Init()` | `MX_WIFI_Init()` → SPI init → module firmware handshake → associate to AP (SSID/password from provisioned flash region) |
| `NET_Connect(host, port)` | Open a TLS socket on the module; trigger TLS handshake; validate server cert against embedded CA |
| `NET_Send(buf, len)` | `MX_WIFI_Socket_send()` — data passed to module over SPI; module transmits over WiFi |
| `NET_Recv(buf, len, ...)` | `MX_WIFI_Socket_recv()` — polls module RX buffer; INTn interrupt used to wake MCU |
| `NET_Disconnect()` | `MX_WIFI_Socket_close()` |

### 4.3 Notes

- WiFi credentials (SSID + password) are stored provisioned in `SRAM3_S` (Secure SRAM). Never in plaintext flash.
- The EMW3080B maintains its own DHCP client and ARP table — the STM32U585 sees only socket-level API.
- 2.4 GHz co-channel interference with the Zigbee node link is mitigated by Zigbee channel planning (ch 25/26) and staggered TX windows. See [[3.5-network-module-selection]] §Option A.

## 5. Implementation B — Ethernet (WIZnet W5500)

### 5.1 Overview

| Property | Value |
|---|---|
| File | `drivers/net/ethernet/eth_driver.c` |
| Compiled when | `-DNET_TRANSPORT_ETHERNET` |
| Underlying library | WIZnet ioLibrary for STM32 HAL |
| Hardware | WIZnet W5500 breakout — wired to SPI2, dedicated chip-select |
| TCP/IP stack | Hardware TCP/IP inside W5500 (8 independent sockets) |
| TLS | mbedTLS running on STM32U585 — TLS record layer uses W5500 socket for TX/RX |

### 5.2 Mapping to `net_api.h`

| `net_api.h` call | W5500 implementation |
|---|---|
| `NET_Init()` | SPI2 init → W5500 reset (RSTn GPIO) → `wizchip_init()` → `DHCP_run()` until lease obtained |
| `NET_Connect(host, port)` | `socket()` → `connect()` via ioLibrary; then mbedTLS handshake using W5500 socket as transport |
| `NET_Send(buf, len)` | mbedTLS write → `send()` via ioLibrary → SPI frame to W5500 |
| `NET_Recv(buf, len, ...)` | mbedTLS read → `recv()` via ioLibrary → SPI read from W5500; INTn interrupt signals packet ready |
| `NET_Disconnect()` | mbedTLS close_notify → `disconnect()` → `close()` |

### 5.3 SPI and GPIO Assignment

| Signal | STM32U585 pin | W5500 pin | Notes |
|---|---|---|---|
| SPI_SCK | (custom PCB assignment) | SCLK | Up to 80 MHz |
| SPI_MOSI | (custom PCB assignment) | MOSI | |
| SPI_MISO | (custom PCB assignment) | MISO | |
| GPIO (CS) | (custom PCB assignment) | SCSn | Active low, dedicated |
| GPIO | (custom PCB assignment) | RSTn | Hardware reset |
| GPIO (EXTI) | (custom PCB assignment) | INTn | Packet-received interrupt → wakes MCU from WFI |

SPI bus is **exclusive to the W5500** — not shared with other peripherals. Enforced in the BSP peripheral ownership table (see [[5.02-memory-map]]).

### 5.4 Notes

- W5500 hardware TCP/IP handles ARP, IP, TCP/UDP — the STM32U585 never processes raw Ethernet frames.
- With Ethernet there is **no 2.4 GHz RF emission** from the gateway's uplink path, eliminating Zigbee co-interference.
- PoE option: W5500-PoE breakout variant allows the gateway to receive both data and power over a single Ethernet cable (802.3af/at PD).

## 6. TLS Certificate Pinning

Both implementations validate the server TLS certificate against a CA certificate embedded in gateway firmware at build time:

```c
/* drivers/net/net_tls.h */
extern const uint8_t NET_CA_CERT_PEM[];   /* Embedded at link time — provisioned per deployment */
extern const size_t  NET_CA_CERT_LEN;
```

- **Phase 1 (EMW3080B):** CA cert is transferred to module during `NET_Init()` via the STM32_Network_Library TLS configuration API. TLS verification runs on the module.
- **Phase 2 (W5500):** CA cert is stored in flash and loaded into mbedTLS on the STM32U585. Verification runs on the host MCU.

The token used for bearer authentication (`Authorization: Bearer <gateway_token>`) is stored in `SRAM3_S` (Secure SRAM) and never exposed to untrusted memory regions.

## 7. Error Handling

| Error | Code | Behaviour |
|---|---|---|
| `NET_ERR_CONNECT` | -1 | CPM retains server TX queue; retries `NET_Connect()` after `NET_RECONNECT_DELAY_MS` |
| `NET_ERR_TLS` | -4 | Log fault; trigger module hardware reset; re-init sequence |
| `NET_ERR_SEND` | -2 | Retain event in server TX queue; retry on next `CPM_Server_Tick()` |
| `NET_ERR_NOT_READY` | -6 | Fatal — re-enter `NET_Init()` sequence |

Module hardware reset (via RSTn GPIO) is the recovery path for persistent `NET_ERR_CONNECT` or `NET_ERR_TLS` after 3 retries. Fault is logged to the fault ring.
