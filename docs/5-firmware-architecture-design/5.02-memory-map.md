## 1. Purpose

This document defines every flash region and SRAM bank for the STM32U585AII6Q, together with all TrustZone security attributes: SECWM option bytes, SAU region table, GTZC configuration, HDP/WRP write protection, RDP level, and the Secure and Non-Secure MPU region tables. The linker memory constants in §9 are the sole authoritative source for all three linker scripts.

## 2. Target MCU Summary

| Property | Value |
|----------|-------|

| CPU | Arm Cortex-M33 (r0p4), TrustZone enabled |
| Flash | 2 MB — 2 banks × 128 pages × 8 KB |
| SRAM total | 784 KB (SRAM1 192 KB + SRAM2 64 KB + SRAM3 512 KB + SRAM4 16 KB) |
| Reference manual | RM0456 Rev 5 |
| TrustZone option byte | `TZEN = 1` (set in production and during development) |

## 3. Flash Layout

### 3.1 Region Table

| Region | Start | End | Size | Bank/Pages | Security | Purpose |
|--------|-------|-----|------|-----------|----------|---------|
| Bootloader | `0x08000000` | `0x0801DFFF` | 120 KB | Bank 1, pages 0–14 | **SECURE** | Bootloader code, ECDSA validation, SAU/GTZC/MPU config |
| Veneer (NSC) | `0x0801E000` | `0x0801FFFF` | 8 KB | Bank 1, page 15 | **NSC** | Non-Secure Callable gateway functions (SG instructions) |
| App A (primary) | `0x08020000` | `0x080FFFFF` | 896 KB | Bank 1, pages 16–127 | **NS** | Active application image |
| App B (OTA staging) | `0x08100000` | `0x081DFFFF` | 896 KB | Bank 2, pages 0–111 | **NS** | OTA download slot; validated before boot |
| Secure Config | `0x081E0000` | `0x081E1FFF` | 8 KB | Bank 2, page 112 | **SECURE** | WiFi credentials, gateway token, Zigbee NWK key |
| OTA Control | `0x081E2000` | `0x081E3FFF` | 8 KB | Bank 2, page 113 | **NS** | Swap directive, pending-update flag, target version |
| Reserved | `0x081E4000` | `0x081FDFFF` | 104 KB | Bank 2, pages 114–126 | NS | Future use |
| Fault Log | `0x081FE000` | `0x081FFFFF` | 8 KB | Bank 2, page 127 | **NS** | 4 KB `fault_record_t` ring buffer (`fault_code_t` + CFSR + PC + LR) |

**Total: 2,048 KB (2 MB)** — both banks fully allocated.

### 3.3 Boot Slot Selection

The bootloader reads the **OTA Control** region at `0x081E2000`. If `swap_pending == 0x5AA5AA5AUL` and `crc_valid == 0x01U`, the bootloader validates App B and jumps there; otherwise it jumps to App A. Failed validation keeps App A and clears the OTA Control flag.

## 4. SRAM Layout

| Bank | Start | End | Size | Security | Usage |
|------|-------|-----|------|----------|-------|
| SRAM1 | `0x20000000` | `0x2002FFFF` | 192 KB | **NS** (GTZC MPCBB) | App `.data`, `.bss`, interrupt stacks |
| SRAM2 | `0x20030000` | `0x2003FFFF` | 64 KB | **NS** (GTZC MPCBB) | CPM TX/RX queues, sensor ring buffers |
| SRAM3 | `0x20040000` | `0x200BFFFF` | 512 KB | **SECURE** (GTZC MPCBB) | AES working buffers, WiFi credentials, gateway token, Zigbee NWK key |
| SRAM4 | `0x28000000` | `0x28003FFF` | 16 KB | **NS** (power domain) | Low-power retained: RTC shadow, occupancy state, last wake reason |

**SRAM3 security note:** SRAM3 is Secure via the GTZC MPCBB (Memory Protection Controller Block-Based). All 512 KB of SRAM3 blocks are set to Secure (block size = 256 bytes). The SAU has no NS entry covering `0x20040000`–`0x200BFFFF`, making it implicitly Secure.

**SRAM4 retention:** SRAM4 remains powered during STOP2 mode. The app writes a validated `wake_state_t` struct there before sleeping and reads it on wake to skip full reinitialisation.

## 5. SECWM — Security Watermark Configuration

SECWM option bytes define the Secure flash page ranges. They are programmed once and verified by the bootloader on every boot.

| Register | Value | Meaning |
|----------|-------|---------|
| `FLASH_SECWM1R1.STRT` | `0` | Bank 1 Secure region starts at page 0 |
| `FLASH_SECWM1R1.END` | `15` | Bank 1 Secure region ends at page 15 (inclusive = 128 KB bootloader) |
| `FLASH_SECWM2R1.STRT` | `112` | Bank 2 Secure region starts at page 112 |
| `FLASH_SECWM2R1.END` | `112` | Bank 2 Secure region ends at page 112 (8 KB Secure Config) |

Any mismatch detected at runtime triggers `fault_record_t` write to Fault Log and `NVIC_SystemReset()`.

## 6. SAU — Security Attribution Unit

**SAU_CTRL.ALLNS = 0** — all memory is Secure by default; only explicitly listed regions are NS or NSC.

| Region # | Base | Limit | Attribute | Description |
|----------|------|-------|-----------|-------------|
| 0 | `0x0801E000` | `0x0801FFFF` | **NSC** | Veneer region — SG instructions, NS→S call gates |
| 1 | `0x08020000` | `0x081DFFFF` | **NS** | App A + App B OTA staging flash |
| 2 | `0x081E2000` | `0x081FFFFF` | **NS** | OTA Control + Reserved + Fault Log |
| 3 | `0x20000000` | `0x2003FFFF` | **NS** | SRAM1 + SRAM2 |
| 4 | `0x28000000` | `0x28003FFF` | **NS** | SRAM4 (low-power retained) |
| 5 | `0x40000000` | `0x4FFFFFFF` | **NS** | Non-Secure peripheral bus |

**Implicitly Secure (not covered by any NS SAU region):**

| Range | Description |
|-------|-------------|
| `0x08000000`–`0x0801DFFF` | Bootloader code (112 KB, pages 0–14) |
| `0x081E0000`–`0x081E1FFF` | Secure Config flash |
| `0x20040000`–`0x200BFFFF` | SRAM3 (managed by GTZC MPCBB) |
| `0x50000000`–`0x5FFFFFFF` | Secure peripheral bus (AES, PKA, RNG, GTZC, FLASH registers, etc.) |

All SAU base and limit addresses are 32-byte aligned per ARMv8-M requirement.

## 7. Write and Hide Protection (WRP / HDP)

| Protection | Register | Setting | Effect |
|-----------|----------|---------|--------|
| Write-Protect | `FLASH_WRP1AR.WRP1A_STRT = 0, WRP1A_END = 15` | Bank 1 pages 0–15 | Prevents erase/write of bootloader (including veneer) from both Secure and NS world |
| Hide Protection | `FLASH_HDP1R.HDPSTRT = 0, HDPEND = 15` | Bank 1 pages 0–15 | Prevents read-out of bootloader pages via debugger or NS code; execution from Secure world is unaffected |
| TrustZone enable | Option byte `TZEN = 1` | Chip-wide | Activates SAU, SECWM, Secure/NS separation |
| RDP level | Option byte `RDP` | `0xAA` (RDP0) dev / `0xBB` (RDP1) production | RDP1 disables debug access to NS world; full chip erase required to regress |

## 8. MPU Regions

### 8.1 Secure MPU (bootloader and Secure services)

Configured in `bsp_secure.c` before branching to the Non-Secure application. Region 15 is the background region (deny-all catch).

| Region | Base | Size | AP | XN | Purpose |
|--------|------|------|----|----|---------|
| 0 | `0x08000000` | 128 KB | **RO/priv** | No (execute) | Bootloader flash — read + execute only |
| 1 | `0x20040000` | 512 KB | **RW/priv** | Yes | SRAM3 — Secure data (keys, credentials) |
| 2 | `0x081E0000` | 8 KB | **RO/priv** | Yes | Secure Config — data only, never execute |
| 3 | `0x50000000` | 256 MB | **RW/priv** | Yes | Secure peripheral bus (AES/PKA/RNG/GTZC/FLASH registers) |
| 15 | `0x00000000` | 4 GB | **No access** | Yes | Background — deny all unmatched accesses |

AP legend: `RO/priv` = read-only, privileged access only. `RW/priv` = read-write, privileged access only.

### 8.2 Non-Secure MPU (application world)

Configured by `BTL_JumpToApp()` in `btl_jump.c` before executing `BLXNS`. Region 15 is the background deny-all.

| Region | Base | Size | AP | XN | Purpose |
|--------|------|------|----|----|---------|
| 0 | `0x08020000` | 896 KB | **RX/any** | No (execute) | App A flash — execute only; write forbidden (prevents self-modification) |
| 1 | `0x08100000` | 896 KB | **RW/any** | Yes | App B OTA staging — write for download; execute prohibited until validated |
| 2 | `0x081E2000` | 24 KB | **RW/any** | Yes | OTA Control + Reserved + Fault Log |
| 3 | `0x20000000` | 192 KB | **RW/any** | Yes | SRAM1 — app data, BSS, main stack |
| 4 | `0x20030000` | 64 KB | **RW/any** | Yes | SRAM2 — queues and ring buffers |
| 5 | `0x28000000` | 16 KB | **RW/any** | Yes | SRAM4 — low-power retained state |
| 6 | *(stack bottom – 32 B)* | 32 B | **No access** | Yes | Stack overflow guard (detect PSP underrun) |
| 7 | `0x40000000` | 256 MB | **RW/any** | Yes | NS peripheral bus (I2C, SPI, USART, GPIO, timers, etc.) |
| 15 | `0x00000000` | 4 GB | **No access** | Yes | Background — deny all unmatched accesses |

AP legend: `RX/any` = read + execute, any privilege. `RW/any` = read-write, any privilege.
XN = Execute Never. App B's XN prevents accidental execution of an unvalidated OTA image.

## 9. Linker Memory Constants

The following symbols are defined in each linker script (`node.ld`, `gateway.ld`, `bootloader.ld`). All three files share the same MEMORY block; targets select what to place in each region.

```ld
MEMORY
{
    FLASH_BOOT   (rx)  : ORIGIN = 0x08000000, LENGTH = 128K   /* Secure bootloader  */
    FLASH_VENEER (rx)  : ORIGIN = 0x0801E000, LENGTH = 8K     /* NSC veneer page    */
    FLASH_APP_A  (rx)  : ORIGIN = 0x08020000, LENGTH = 896K   /* Primary app slot   */
    FLASH_APP_B  (rw)  : ORIGIN = 0x08100000, LENGTH = 896K   /* OTA staging slot   */
    FLASH_SCFG   (r)   : ORIGIN = 0x081E0000, LENGTH = 8K     /* Secure Config      */
    FLASH_OTACTL (rw)  : ORIGIN = 0x081E2000, LENGTH = 8K     /* OTA Control        */
    FLASH_FLTLOG (rw)  : ORIGIN = 0x081FE000, LENGTH = 8K     /* Fault Log          */

    SRAM1        (rwx) : ORIGIN = 0x20000000, LENGTH = 192K   /* App data + stacks  */
    SRAM2        (rw)  : ORIGIN = 0x20030000, LENGTH = 64K    /* Queues + buffers   */
    SRAM3        (rw)  : ORIGIN = 0x20040000, LENGTH = 512K   /* Secure data        */
    SRAM4        (rw)  : ORIGIN = 0x28000000, LENGTH = 16K    /* Low-power retained */
}
```

Symbolic aliases used in C source (defined in `bsp.h`):

```c
#define BSP_FAULT_LOG_BASE  0x081FE000UL
#define BSP_FAULT_LOG_SIZE  0x00001000UL   /* 4 KB ring (half the page) */
#define BSP_OTA_CTRL_BASE   0x081E2000UL
#define BSP_APP_A_BASE      0x08020000UL
#define BSP_APP_B_BASE      0x08100000UL
#define BSP_SCFG_BASE       0x081E0000UL
```
