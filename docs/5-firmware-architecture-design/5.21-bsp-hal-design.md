## 1. Purpose

This document defines how the STM32U585AII6Q hardware resources are owned and initialised. It covers: peripheral ownership, clock tree, GPIO pin assignments, the complete Secure-world init sequence (SAU, GTZC, Secure MPU), the Non-Secure MPU configuration, the veneer function catalog for NS→S calls, and the STOP2 low-power entry/exit procedure.

The BSP has two compilation units: `bsp.c` (Non-Secure, compiled into Node/Gateway app) and `bsp_secure.c` (Secure, compiled into all three targets but linked only into the Secure flash region). The bootloader calls `bsp_secure.c` functions directly; the application calls through the NSC veneer.

## 3. Clock Tree

Target: **SYSCLK = 160 MHz** from PLL1, sourced from the internal HSI16.

```
HSI16 (16 MHz)
  └──▶ PLL1
         PLLM = 1   → VCO input = 16 MHz
         PLLN = 10  → VCO output = 160 MHz
         PLLP = 1   → PLLP_CLK = 160 MHz  → SYSCLK
         PLLQ = 2   → PLLQ_CLK = 80 MHz   → USB / RNG input
         PLLR = 1   → PLLR_CLK = 160 MHz  → reserved

SYSCLK = 160 MHz
  ├── AHB1/2/3  (HCLK   = 160 MHz, prescaler = 1)
  ├── APB1      (PCLK1  =  80 MHz, prescaler = 2)
  ├── APB2      (PCLK2  = 160 MHz, prescaler = 1)
  └── APB3      (PCLK3  =  80 MHz, prescaler = 2)

Low-power clocks (always-on):
  LSE (32.768 kHz)  → RTC, LPTIM1 in STOP2
  LSI  (32 kHz)     → IWDG (fallback if LSE fails)
```

Flash latency for 160 MHz at VCORE range 1 (1.2 V): **latency = 4 wait states** (`FLASH_ACR.LATENCY = 4`). Set before switching PLL, cleared after in case of clock step-down.

## 5. BSP Secure Initialisation Sequence

`bsp_secure.c` provides `BSP_Secure_Init()`, called first in `btl_main.c` (bootloader) and by the Secure startup code in application targets before handing off to NS. The sequence is fixed and must not be reordered.

```
BSP_Secure_Init()
│
├── 1.  HAL_Init()                         Set SysTick to 1 ms; enable instruction cache
│
├── 2.  Flash latency                       FLASH_ACR.LATENCY = 4 (required for 160 MHz)
│
├── 3.  PLL configuration                   HSI16 → PLL1 (×10) → 160 MHz SYSCLK
│       SystemCoreClockUpdate()
│
├── 4.  Peripheral clock enables            AHB/APB gates for SRAM, GPIO, AES, PKA, RNG,
│                                           GTZC, I2C, SPI, USART, IPCC, IWDG, TIM2, LPTIM1
│
├── 5.  Verify SECWM option bytes           Read FLASH_SECWM1R1 (STRT=0, END=15)
│                                           Read FLASH_SECWM2R1 (STRT=112, END=112)
│       On mismatch:  → write fault_record_t to Fault Log → NVIC_SystemReset()
│
├── 6.  SAU configuration                   Disable SAU while updating (SAU_CTRL.ENABLE = 0)
│       Region 0: 0x0801E000–0x0801FFFF  NSC  (veneer)
│       Region 1: 0x08020000–0x081DFFFF  NS   (App A + App B)
│       Region 2: 0x081E2000–0x081FFFFF  NS   (OTA Ctrl + Reserved + Fault Log)
│       Region 3: 0x20000000–0x2003FFFF  NS   (SRAM1 + SRAM2)
│       Region 4: 0x28000000–0x28003FFF  NS   (SRAM4)
│       Region 5: 0x40000000–0x4FFFFFFF  NS   (NS peripheral bus)
│       SAU_CTRL.ALLNS = 0, SAU_CTRL.ENABLE = 1
│       __DSB(); __ISB();
│
├── 7.  GTZC1 configuration
│       MPCBB1 (SRAM1): all 768 blocks → NS
│       MPCBB2 (SRAM2): all 256 blocks → NS
│       MPCBB3 (SRAM3): all 2048 blocks → SECURE
│       MPCBB5 (SRAM4): all 64 blocks → NS
│       Peripheral TZ (PRIVCFGR/SECCFGR): AES, PKA, RNG → SECURE
│                                          I2C1, SPI2, USART1, IWDG → NS
│
├── 8.  Secure MPU configuration            __set_PRIMASK(1)  (disable interrupts during config)
│       Region 0: 0x08000000  128 KB  RO+exec   priv    Bootloader flash
│       Region 1: 0x20040000  512 KB  RW+noexec priv    SRAM3 Secure data
│       Region 2: 0x081E0000    8 KB  RO+noexec priv    Secure Config flash
│       Region 3: 0x50000000  256 MB  RW+noexec priv    Secure peripheral bus
│       Region 15: 0x00000000   4 GB  no-access  bg     Background deny
│       MPU_CTRL = MPU_CTRL_ENABLE | MPU_CTRL_PRIVDEFENA; __DSB(); __ISB()
│       __set_PRIMASK(0)
│
├── 9.  LSE + RTC init                      Enable LSE; configure RTC for wake timestamp
│
├── 10. IWDG start                          LSI clock, 4 s timeout, prescaler PR=64
│
└── 11a. [Bootloader only]
│       btl_validate.c: validate App A (or App B if swap_pending)
│       btl_jump.c: configure NS MPU, set VTOR_NS, BLXNS to NS reset handler
│
    11b. [Application Secure startup]
        Configure NS MPU (§8 of this doc)
        → return to NS startup code (app)
```

## 6. Secure MPU Region Definitions (C register values)

These are applied in step 8 above. M33 MPU uses RBAR (base + attributes) and RLAR (limit + enable + attribute index). ARM Cortex-M33 uses the PMSAv8 model — arbitrary base/limit, no power-of-2 alignment requirement.

```c
/* Region 0 — Bootloader flash: RX, privileged */
MPU->RNR  = 0U;
MPU->RBAR = (0x08000000UL & MPU_RBAR_BASE_Msk)    /* Base  */
           | (3UL  << MPU_RBAR_AP_Pos)              /* AP=11: RO, priv-only */
           | (0UL  << MPU_RBAR_XN_Pos);             /* XN=0: execute allowed */
MPU->RLAR = ((0x0801FFFFUL - 1U) & MPU_RLAR_LIMIT_Msk)  /* Limit */
           | (0UL  << MPU_RLAR_AttrIndx_Pos)        /* Attr 0: Normal WB */
           | MPU_RLAR_EN_Msk;

/* Region 1 — SRAM3: RW, privileged, no execute */
MPU->RNR  = 1U;
MPU->RBAR = (0x20040000UL & MPU_RBAR_BASE_Msk)
           | (1UL  << MPU_RBAR_AP_Pos)              /* AP=01: RW, priv-only */
           | (1UL  << MPU_RBAR_XN_Pos);             /* XN=1: no execute */
MPU->RLAR = ((0x200BFFFFUL - 1U) & MPU_RLAR_LIMIT_Msk)
           | (0UL  << MPU_RLAR_AttrIndx_Pos)
           | MPU_RLAR_EN_Msk;

/* Region 2 — Secure Config: RO, privileged, no execute */
MPU->RNR  = 2U;
MPU->RBAR = (0x081E0000UL & MPU_RBAR_BASE_Msk)
           | (3UL  << MPU_RBAR_AP_Pos)
           | (1UL  << MPU_RBAR_XN_Pos);
MPU->RLAR = ((0x081E1FFFUL - 1U) & MPU_RLAR_LIMIT_Msk)
           | (0UL  << MPU_RLAR_AttrIndx_Pos)
           | MPU_RLAR_EN_Msk;

/* Region 3 — Secure peripheral bus: RW, privileged, no execute */
MPU->RNR  = 3U;
MPU->RBAR = (0x50000000UL & MPU_RBAR_BASE_Msk)
           | (1UL  << MPU_RBAR_AP_Pos)
           | (1UL  << MPU_RBAR_XN_Pos);
MPU->RLAR = ((0x5FFFFFFFUL - 1U) & MPU_RLAR_LIMIT_Msk)
           | (1UL  << MPU_RLAR_AttrIndx_Pos)        /* Attr 1: Device nGnRE */
           | MPU_RLAR_EN_Msk;

/* Region 15 — Background: no access (explicit deny for unmatched regions) */
MPU->RNR  = 15U;
MPU->RBAR = (0x00000000UL & MPU_RBAR_BASE_Msk)
           | (0UL  << MPU_RBAR_AP_Pos)               /* AP=00: no access */
           | (1UL  << MPU_RBAR_XN_Pos);
MPU->RLAR = ((0xFFFFFFFFUL - 1U) & MPU_RLAR_LIMIT_Msk)
           | (0UL  << MPU_RLAR_AttrIndx_Pos)
           | MPU_RLAR_EN_Msk;

/* Attribute index 0 = Normal, Write-Back, Read-Allocate, Write-Allocate */
MPU->MAIR0 = 0xFF000000UL;   /* Attr 0: MAIR = 0xFF (Normal WB RA WA) */
/* Attribute index 1 = Device, nGnRE */
MPU->MAIR0 |= 0x000000040UL; /* Attr 1: MAIR = 0x04 (Device nGnRE) */

MPU->CTRL  = MPU_CTRL_ENABLE_Msk | MPU_CTRL_PRIVDEFENA_Msk;
__DSB();
__ISB();
```

## 7. Non-Secure MPU Region Definitions

Configured by `BTL_JumpToApp()` in `btl_jump.c` **before** the `BLXNS` instruction. Uses the Non-Secure MPU alias (`MPU_NS`).

| Region | Base | Limit | AP | XN | Description |
|--------|------|-------|----|----|-------------|
| 0 | `0x08020000` | `0x080FFFFF` | RX, any | No | App A — execute; write forbidden |
| 1 | `0x08100000` | `0x081DFFFF` | RW, any | Yes | App B — OTA write; never execute |
| 2 | `0x081E2000` | `0x081FFFFF` | RW, any | Yes | OTA Ctrl + Reserved + Fault Log |
| 3 | `0x20000000` | `0x2002FFFF` | RW, any | Yes | SRAM1 — data, BSS, main stack |
| 4 | `0x20030000` | `0x2003FFFF` | RW, any | Yes | SRAM2 — queues and buffers |
| 5 | `0x28000000` | `0x28003FFF` | RW, any | Yes | SRAM4 — low-power retained |
| 6 | `(PSP_top – 32)` | `(PSP_top – 1)` | No access | Yes | Stack guard — 32-byte trap below PSP |
| 7 | `0x40000000` | `0x4FFFFFFF` | RW, any | Yes | NS peripheral bus |
| 15 | `0x00000000` | `0xFFFFFFFF` | No access | Yes | Background deny-all |

AP definitions: `RX, any` = AP=10 (read-only, any privilege, execute allowed). `RW, any` = AP=11-like but XN set. No-access = AP=00.

## 8. Veneer Function Catalog (NS→Secure Call Gates)

The veneer region (`0x0801E000`–`0x0801FFFF`, page 15, NSC) contains all SG instruction entry points. NS code calls these functions as ordinary C function pointers; the processor transitions to Secure state at the SG instruction.

| Veneer function | Signature | Purpose |
|----------------|-----------|---------|
| `SEC_RNG_GetRandom` | `ps_status_t SEC_RNG_GetRandom(uint8_t *p_buf, uint16_t len_bytes)` | Fill `p_buf` with `len_bytes` TRNG bytes from the Secure RNG peripheral. Plaintext never written to NS memory by Secure code itself; caller provides the NS buffer pointer which is validated before use. |
| `SEC_KeyStore_Init` | `ps_status_t SEC_KeyStore_Init(void)` | Read credentials from Secure Config flash (`0x081E0000`) into SRAM3 key table. Called once at startup before any CPM transmission. |
| `SEC_KeyStore_GetEncryptedPayload` | `ps_status_t SEC_KeyStore_GetEncryptedPayload(const cpm_plaintext_t *p_in, cpm_ciphertext_t *p_out)` | AES-128-CCM encrypt CPM payload using stored NWK key. The NWK key never leaves SRAM3; only ciphertext written to the NS-accessible `p_out` buffer. |
| `SEC_Flash_WriteConfig` | `ps_status_t SEC_Flash_WriteConfig(const uint8_t *p_data, uint16_t len)` | Erase and reprogram Secure Config page at `0x081E0000` with validated `p_data`. Length must equal `sizeof(secure_config_t)`. Validates magic + CRC-32 before writing. |

**Security invariants enforced in every veneer function:**
1. Validate all NS pointer arguments with `cmse_check_address_range()` before dereferencing — reject any pointer into Secure address space.
2. Copy input data from NS to a Secure-stack local buffer before processing.
3. Zeroize Secure-stack locals containing key material before return.
4. Return only `ps_status_t`; never return raw key bytes or pointers to SRAM3.

## 9. Low-Power Entry and Exit (STOP2 — Node only)

The Node enters STOP2 between sensing cycles to achieve the 10 µA target (spec NFR-PWR-001).

### 9.1 Entry Sequence (`BSP_EnterStop2()`)

```
1. PDM_GetOccupancy()      → write result to SRAM4 wake_state.last_occupancy
2. CPM_Flush()             → block until TX queue empty (or timeout 500 ms)
3. BSP_IWDG_Pet()          → reset watchdog before sleep
4. Configure LPTIM1        → set compare for 30 s on LSE, enable LPTIM1 wake EXTI
5. HAL_SuspendTick()       → stop SysTick; TIM2 tick source gated
6. __set_PRIMASK(1)        → disable interrupts during sleep prep
7. __DSB(); __WFI()        → enter STOP2 (VCORE off, regulated domain powered down)
```

CPU stalls at `__WFI()` until LPTIM1 compare event or EXTI (button/sensor INT).

### 9.2 Exit / Wakeup Sequence

```
1. Hardware resumes at STOP2 exit handler (LPTIM1 or EXTI ISR)
2. Restore SYSCLK          → re-enable PLL1, wait PLLRDY, switch SYSCLK
3. HAL_ResumeTick()        → re-enable TIM2 tick
4. __set_PRIMASK(0)
5. BSP_IWDG_Pet()          → pet watchdog before re-entering super loop
6. Read SRAM4 wake_state.wake_reason
   IF wake_reason == WAKE_SENSOR_INT  → immediate PDM_Update() cycle
   IF wake_reason == WAKE_TIMER       → normal sensing cycle
7. I2C1 re-init            → HAL_I2C_Init() (peripheral resets on STOP2 exit)
8. IPCC re-init            → RF_Resume() (RF co-processor not affected by STOP2)
```

Peripheral state after STOP2: GPIO and IWDG state retained. I2C, SPI, USART state NOT retained — drivers must call their HAL init functions on wake. The IPCC channel to the STM32WB RF co-processor is persistent across STOP2.
