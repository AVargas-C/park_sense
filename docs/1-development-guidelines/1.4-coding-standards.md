## Standard: BARR-C:2018 (Embedded C Coding Standard)

This project follows the **[Barr Group Embedded C Coding Standard](https://barrgroup.com/embedded-systems/books/embedded-c-coding-standard)**, which is purpose-built for safety-critical and resource-constrained embedded systems.

---

## Core Rules

### 1. General

- **C Standard:** C11 (`-std=c11`), with compiler extensions disabled (`-pedantic`).
- **Warnings:** Build with `-Wall -Wextra -Werror`. Zero warnings policy.
- **No dynamic memory:** `malloc()`, `calloc()`, `realloc()`, `free()` are **prohibited** in firmware. All memory is statically allocated.
- **No recursion:** Stack depth must be deterministic.

### 2. Braces & Indentation

- **Always use braces**, even for single-line `if`/`else`/`for`/`while` bodies.
- **Allman style** (opening brace on its own line).
- **4 spaces** for indentation. No tabs.

```c
if (sensor_ready)
{
    status = sensor_read(&distance_mm);
}
else
{
    status = SENSOR_ERR_NOT_READY;
}
```

### 3. Line Length & Whitespace

- Maximum **100 characters** per line.
- One blank line between function definitions.
- No trailing whitespace.
- One space after keywords (`if`, `for`, `while`, `switch`, `return`).
- No space between function name and opening parenthesis.

### 4. Variable Declarations

- Declare variables at the top of the block scope.
- One declaration per line.
- Always initialize variables at declaration.

```c
void parking_detect_update(void)
{
    uint16_t distance_mm = 0U;
    sensor_status_t status = SENSOR_OK;

    status = sensor_read(&distance_mm);
    /* ... */
}
```

### 5. Integer Types

- Use **fixed-width types** from `<stdint.h>` exclusively: `uint8_t`, `int16_t`, `uint32_t`, etc.
- Use `bool` from `<stdbool.h>` for boolean values.
- Append `U` suffix to unsigned literals: `0U`, `0xFFU`, `1000U`.

### 6. Constants & Literals

- No magic numbers. Use `#define` or `enum` for all constants.
- Hex constants use uppercase: `0xDEADBEEF`.
- Binary register masks documented inline.

```c
#define SENSOR_I2C_ADDR         (0x29U)   /* device-specific: check datasheet */
#define DETECTION_THRESHOLD_MM  (200U)

/* SPI_BASE->CR1 register: bit 0 = enable, bit 3 = interrupt enable */
/* (Register names are MCU-specific — always reference the datasheet) */
SPI_BASE->CR1 |= (1U << 0U) | (1U << 3U);
```

### 7. Functions

- Functions should do **one thing** and be ≤ 75 lines (excluding comments and braces).
- All functions must have an explicit return type (no implicit `int`).
- `static` for all file-scoped functions (internal linkage).
- Always check return values from functions that can fail.

```c
static sensor_status_t read_distance(uint16_t *p_distance_mm)
{
    if (NULL == p_distance_mm)
    {
        return SENSOR_ERR_NULL_PTR;
    }
    /* ... */
}
```

### 8. Pointers

- Yoda conditions for pointer checks: `NULL == ptr` (prevents accidental assignment).
- Pointer parameters that are not modified must be `const`.
- No pointer arithmetic for array traversal unless performance-critical and documented.

### 9. `volatile` & Hardware Access

- All hardware registers accessed through `volatile`-qualified pointers.
- Shared variables between ISR and main context must be `volatile`.
- Accesses to multi-byte volatile variables must be atomic or protected.

### 10. `switch` Statements

- Every `case` ends with `break`, `return`, or an explicit `/* fall through */` comment.
- Always include a `default` case.

```c
switch (state)
{
    case STATE_IDLE:
        enter_sleep();
        break;

    case STATE_DETECT:
        run_detection();
        break;

    default:
        error_handler();
        break;
}
```

### 11. Preprocessor

- Use `#include` guards in every header (not `#pragma once` for portability).
- Macro names in `UPPER_SNAKE_CASE`.
- Parenthesize all macro arguments and the entire macro body.

```c
#ifndef HAL_SPI_H
#define HAL_SPI_H

#define MAX(a, b)  (((a) > (b)) ? (a) : (b))

#endif /* HAL_SPI_H */
```

## Compiler Flags Summary

```cmake
target_compile_options(firmware PRIVATE
    -std=c11
    -pedantic
    -Wall
    -Wextra
    -Werror
    -Wshadow
    -Wdouble-promotion
    -Wformat=2
    -Wconversion
    -fno-common
    -ffunction-sections
    -fdata-sections
)
```

## Quick Reference

| Rule | Standard |
|---|---|
| Braces | Allman, always use braces |
| Indentation | 4 spaces, no tabs |
| Line length | ≤ 100 characters |
| Integer types | `<stdint.h>` fixed-width only |
| Dynamic memory | Prohibited |
| Recursion | Prohibited |
| Warnings | `-Wall -Wextra -Werror`, zero warnings |
| Function length | ≤ 75 lines |
| Pointer checks | Yoda conditions (`NULL == ptr`) |
| Include guards | `#ifndef` / `#define` / `#endif` |
