## Tool: Doxygen

All C source and header files are documented using **[Doxygen](https://www.doxygen.nl/)** syntax with the `@` command style.

---

## File Header

Every `.c` and `.h` file begins with a file header block:

```c
/**
 * @file    hal_spi.c
 * @brief   SPI peripheral abstraction for the HAL layer.
 *
 * Provides init, transmit, receive, and transfer functions for
 * SPI peripherals. Board-specific pin mapping is handled by the BSP.
 *
 * @note    Hardware dependency: MCU SPI peripheral (via BSP).
 *
 * @author  Arturo Vargas Cuevas
 * @date    2026-01-24
 *
 * @copyright Copyright (c) 2026 Arturo Vargas Cuevas. All rights reserved.
 * @license   Apache-2.0
 */
```

### Required Fields

| Tag | Purpose |
|---|---|
| `@file` | File name (must match actual file name) |
| `@brief` | One-line summary of the file's purpose |
| `@note` | Hardware dependencies, assumptions, constraints |
| `@author` | Primary author |
| `@date` | Creation date (ISO 8601) |
| `@copyright` | Copyright notice |
| `@license` | License identifier |

## Function Headers

Every **public** function (declared in a header) must have a full Doxygen block. `static` functions require at minimum a `@brief`.

### Full Function Header (Public)

```c
/**
 * @brief   Read distance measurement from the ToF sensor.
 *
 * Triggers a single-shot ranging measurement and returns the
 * distance in millimeters. Blocks until measurement is complete
 * or timeout occurs.
 *
 * @param[out]  p_distance_mm  Pointer to store the measured distance (mm).
 *                             Must not be NULL.
 * @param[in]   timeout_ms     Maximum time to wait for measurement (ms).
 *
 * @return  sensor_status_t
 * @retval  SENSOR_OK             Measurement successful.
 * @retval  SENSOR_ERR_TIMEOUT    Measurement did not complete in time.
 * @retval  SENSOR_ERR_NULL_PTR   p_distance_mm was NULL.
 *
 * @note      Sensor must be initialized via tof_sensor_init() before calling.
 * @warning   Not reentrant. Do not call from ISR context.
 *
 * @see tof_sensor_init()
 */
sensor_status_t tof_sensor_read_distance(uint16_t *p_distance_mm,
                                         uint32_t timeout_ms);
```

### Required Tags by Visibility

| Tag | Public Functions | Static Functions |
|---|---|---|
| `@brief` | Required | Required |
| `@param` | Required (all params) | Optional |
| `@return` / `@retval` | Required | Optional |
| `@note` | When applicable | Optional |
| `@warning` | When applicable | Optional |
| `@see` | Cross-references | Optional |

## Param Direction Tags

| Tag | Meaning |
|---|---|
| `@param[in]` | Input only — caller provides data |
| `@param[out]` | Output only — function writes data |
| `@param[in,out]` | Modified by the function |

## Inline Comments

### Rule: Explain **why**, not **what**

```c
/* BAD — restates the code */
distance_mm = raw_value * 2;  /* Multiply raw value by 2 */

/* GOOD — explains the intent */
distance_mm = raw_value * 2;  /* Datasheet §4.3: raw units are 0.5 mm */
```

### Register Bit Manipulation

Always document register accesses with the register name, bit field, and purpose. Register names are MCU-specific — always reference the device datasheet.

```c
/* SPI_BASE->CR1: Set SPE (bit 6) to enable the SPI peripheral */
SPI_BASE->CR1 |= (1U << 6U);

/* UART_BASE->ISR: Wait for TXE (bit 7) — TX data register empty */
while (0U == (UART_BASE->ISR & (1U << 7U)))
{
    /* Spin until TX buffer is available */
}

/* ADC_BASE->CR: Set ADSTART (bit 2) to begin conversion */
ADC_BASE->CR |= (1U << 2U);
```

### ISR Context

```c
/**
 * @brief  SPI interrupt handler — handles TX complete.
 *
 * Clears the TXE flag and signals the CAL layer via callback.
 * Must remain short — no blocking calls.
 */
void SPI_IRQHandler(void)
{
    /* Clear TXE flag by writing next byte or disabling interrupt */
    SPI_BASE->CR2 &= ~(1U << 7U);  /* CR2: TXEIE (bit 7) — disable TX interrupt */

    s_tx_complete = true;  /* Signal main context */
}
```
