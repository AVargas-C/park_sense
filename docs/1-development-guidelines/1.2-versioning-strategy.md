## Semantic Versioning (SemVer)

This project follows **[Semantic Versioning 2.0.0](https://semver.org/)**.

```
MAJOR.MINOR.PATCH
```

| Segment   | Incremented When                                              | Example           |
| --------- | ------------------------------------------------------------- | ----------------- |
| **MAJOR** | Breaking change to the firmware API or communication protocol | `1.0.0` → `2.0.0` |
| **MINOR** | New feature added in a backward-compatible manner             | `1.0.0` → `1.1.0` |
| **PATCH** | Bug fix, no API change                                        | `1.1.0` → `1.1.1` |
|           |                                                               |                   |

### Pre-release Labels

For development builds before a stable release:

```
1.0.0-alpha.1    → Early development, unstable
1.0.0-beta.1     → Feature-complete, testing in progress
1.0.0-rc.1       → Release candidate, final validation
1.0.0            → Stable release
```

## Git Tags

Every release merged into `main` is tagged:

```bash
git tag -a v1.0.0 -m "Release 1.0.0 — initial stable firmware"
git push origin v1.0.0
```
