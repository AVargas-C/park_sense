## Candidates Evaluated

All candidates are ST VL53-family SPAD-based ToF sensors. Competing non-ST ToF sensors (Broadcom AFBR series, AMS TMF882x) are excluded: they target industrial ranges (>1 m to >10 m), are not present in any equivalent dev kit, and have no ST HAL integration path.

| Feature                      | VL53L0X                      | VL53L1X                      | VL53L5CX                            |
| ---------------------------- | ---------------------------- | ---------------------------- | ----------------------------------- |
| **Generation**               | 1st                          | 2nd                          | 4th                                 |
| **Ranging zones**            | 1 (single-point)             | 1 (single-point)             | 8×8 = 64 zones                      |
| **Max range**                | ~2 m                         | ~4 m                         | ~4 m                                |
| **Ranging mode**             | Phase-based                  | Phase + histogram            | Histogram-based (multi-target)      |
| **Interface**                | I2C only                     | I2C only                     | I2C or SPI                          |
| **Package**                  | LGA-12 (2.4×4.4 mm)         | LGA-12 (2.4×4.4 mm)         | LGA-16L (6.4×3.6 mm)               |
| **XSHUT (HW shutdown)**      | Yes                          | Yes                          | Yes (LPn pin)                       |
| **Active current (typical)** | ~20 mA (ranging)             | ~20 mA (ranging)             | ~50 mA (all zones) / ~4 mA (1 zone, power save) |
| **Sleep / standby current**  | ~5 µA                        | ~5 µA                        | ~5 µA                               |
| **Ambient light immunity**   | Moderate                     | Improved                     | Excellent (histogram rejects noise) |
| **ST ULD / driver**         | VL53L0X API                  | VL53L1X API                  | VL53L5CX ULD (official C driver)    |
| **Unit price (est.)**        | ~$3–5                        | ~$5–7                        | ~$8–12                              |
| **On B-U585I-IOT02A**        | No                           | No                           | **Yes**                             |

## Key Observations

**VL53L0X — adequate but obsolete:**
Fully sufficient for binary occupancy detection: a vehicle's undercarriage at 150–400 mm will be unambiguously detected by a single-zone sensor. The VL53L0X is the workhorse of maker projects for this reason. However, it is a first-generation part with no histogram processing and moderate ambient light rejection. Discontinued in new designs in favour of L1X and L5CX.

**VL53L1X — the viable alternative:**
Direct functional replacement for VL53L0X with longer range, improved ambient light immunity, and histogram-assisted ranging. Same package, same I2C interface, same power profile. For a parking sensor requiring binary occupancy only, the VL53L1X represents the best cost/capability trade-off among standalone ToF sensors.

**VL53L5CX — overkill, but dev-kit justified:**
The 8×8 multi-zone capability is unnecessary for binary parking occupancy — the firmware only needs to determine presence or absence above the sensor, not map 64 distance zones. In active mode with all 64 zones enabled, current draw (~50 mA) exceeds the single-zone parts. However, the VL53L5CX supports a 1-zone power-save mode that reduces current to ~4 mA during ranging, comparable to single-zone alternatives. The histogram-based ranging provides superior ambient light rejection at outdoor distances.

## Trade-Off Summary

| Criterion                      | VL53L0X   | VL53L1X   | VL53L5CX     |
| ------------------------------ | --------- | --------- | ------------ |
| Binary occupancy detection     | ✅         | ✅         | ✅            |
| I2C interface                  | ✅         | ✅         | ✅ (also SPI) |
| HW shutdown (XSHUT / LPn)      | ✅         | ✅         | ✅            |
| Active current (power save)    | ⚠️ (~20 mA) | ⚠️ (~20 mA) | ✅ (~4 mA, 1-zone mode) |
| Ambient light rejection        | ⚠️         | ✅         | ✅            |
| ST ULD driver support          | Legacy     | ✅         | ✅ (actively maintained) |
| On B-U585I-IOT02A dev kit      | ❌         | ❌         | ✅            |
| Unit cost                      | ✅ (lowest) | ✅         | ⚠️ (highest)  |

## Selection Rationale

### Selected: VL53L5CX (VL53L5CXV0GC/1)

**Primary decision driver: included on the B-U585I-IOT02A development kit.**

For binary parking occupancy detection, any ST VL53-family sensor is technically adequate. The VL53L0X or VL53L1X would satisfy SYS-F-002 and SYS-I-030 with no firmware compromise. The VL53L5CX's 64-zone capability is unused in the initial implementation — the firmware reads a single central zone in power-save mode.

**Why VL53L5CX nonetheless:**

| Factor                          | Rationale                                                                                          |
| ------------------------------- | -------------------------------------------------------------------------------------------------- |
| Dev kit integration             | Pre-wired and validated on B-U585I-IOT02A — no external wiring, schematic, or bring-up cost        |
| 1-zone power-save mode          | ~4 mA active current in single-zone mode is competitive with simpler VL53L0X/L1X (~20 mA)         |
| ST ULD driver                   | Official ST Ultra Lite Driver provides clean C API — no porting effort                              |
| Ambient light rejection         | Histogram-based ranging is more robust in outdoor parking environments with sunlight interference   |
| Portfolio depth                 | Demonstrates familiarity with a more capable sensor; single-zone operation documents a deliberate architectural decision |

> **Note:** In a production design optimised for cost, the VL53L1X would be the preferred standalone sensor. The VL53L5CX is selected here because the dev kit provides it at no additional hardware effort.

## Risk Assessment

| Risk Factor               | Assessment                                                                                             |
| ------------------------- | ------------------------------------------------------------------------------------------------------ |
| Supply chain availability | **Low risk.** VL53L5CX is in active production. Available through ST and major distributors.           |
| Driver / API complexity   | **Low risk.** ST ULD is well-documented with sample code. Single-zone usage is the simplest API path.  |
| Documentation quality     | **Low risk.** Full datasheet, application notes, and ULD reference manual from ST.                     |
| Long-term support         | **Low risk.** 4th-generation VL53 family — ST's current mainstream ToF family.                        |
| Dev kit availability      | **Low risk.** B-U585I-IOT02A is readily available from ST and distributors.                            |
