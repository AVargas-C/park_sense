## Key Observations

**IIS2MDCTR — the primary candidate:**
State-of-the-art ST magnetometer with the lowest noise floor (1.0 mGauss RMS) in its class. The ±50 Gauss range accommodates the strong ferromagnetic anomaly of a vehicle body at close range without saturation. Ultra-low 1 µA idle current makes it ideal for duty-cycled battery operation. Available across both I2C and SPI. Directly supported by ST's sensor abstraction libraries. The STEVAL-MKBOXPRO uses the IIS2MDC (same die) confirming ST's own endorsement of this sensor for platform-level evaluation hardware.

**LIS3MDL — viable but legacy:**
Found on the STEVAL-BCN002V1 (LIS3MDL). Adequate range (±16 Gauss maximum) and sensitivity, but higher noise (~3.2 mGauss vs 1.0 mGauss), higher sleep current (~5 µA vs ~1 µA), and lower ODR (80 Hz). ST currently classifies LIS3MDL as a production-only part — no new designs recommended. The noise floor disadvantage is significant for outdoor deployment where ambient magnetic noise is present.

**LIS2MDL — current alternative to IIS2MDCTR:**
Almost identical performance to IIS2MDCTR (same range, same noise, same ODR, same sleep current). LIS2MDL differs primarily in package availability and cost. If the B-U585I-IOT02A were not the chosen platform, LIS2MDL would be the equivalent standalone choice.

**HMC5883L / QMC5883L — eliminated:**
The HMC5883L (Honeywell) is discontinued. The QMC5883L is a clone with a different I2C register map — drivers written for HMC5883L are not compatible. Range limited to ±8 Gauss may saturate in close proximity to a vehicle body. No temperature compensation. No integrated hard-iron offset correction. Insufficient for outdoor professional deployment.

**BMM150 — eliminated:**
Low maximum ODR (10 Hz in normal mode) is a concern for rapid detection. No ST ecosystem support. Bosch SDK is available but adds integration complexity. The range is competitive but the ODR limitation and non-ST ecosystem make it a poor fit for this project.

## Selection Rationale

### Selected: IIS2MDCTR

**Primary decision driver: included on the B-U585I-IOT02A development kit.**

For vehicle ferromagnetic detection, any of the ST magnetometers with ±50 Gauss range and sub-2 mGauss noise floor is technically adequate. The LIS2MDL provides equivalent performance as a standalone alternative. The IIS2MDCTR is selected because it is integrated on the B-U585I-IOT02A board, eliminating external wiring and hardware bring-up cost.

| Factor                          | Rationale                                                                                              |
| ------------------------------- | ------------------------------------------------------------------------------------------------------ |
| Dev kit integration             | Pre-wired and validated on B-U585I-IOT02A — I2C bus connected, address configured, no external parts   |
| ±50 Gauss range                 | No saturation risk from vehicle body magnetic anomaly at close range                                   |
| 1.0 mGauss RMS noise floor      | Best-in-class for outdoor detection; resolves subtle field perturbations in noisy environments          |
| 1 µA idle current               | Negligible contribution to node sleep current budget                                                   |
| Temperature compensation        | Essential for outdoor deployment across the full operating temperature range                            |
| ST driver ecosystem             | Consistent with MCU (STM32U585) and HAL — no cross-vendor driver integration                           |
| STEVAL-MKBOXPRO uses IIS2MDC    | ST's own platform-level evaluation hardware uses the same sensor die — confirms suitability             |

> **Note on packaging:** IIS2MDCTR (T = tape-and-reel) and IIS2MDC (tray) are the same silicon. The B-U585I-IOT02A uses the tape-and-reel variant as assembled by ST. For a custom PCB, IIS2MDC (tray) can be sourced in smaller quantities.
