## Option A � WiFi Module

### Selected: EMW3080B (MXCHIP / ST67W611M1)

The WiFi module selection is **pre-determined by the development kit.** The B-U585I-IOT02A evaluation board includes the EMW3080B soldered directly to the board, pre-wired to the STM32U585 over SPI, with a validated power rail and ST-official firmware. No additional hardware procurement or integration effort is required for the development phase.

#### Why EMW3080B

| Factor                         | Rationale                                                                                                      |
| ------------------------------ | -------------------------------------------------------------------------------------------------------------- |
| Dev kit pre-installed          | No procurement, no wiring � SPI bus and power rails validated by ST out of the box                             |
| ST official driver             | STM32_Network_Library in STM32CubeU5 � no AT-command parsing, clean socket-level API                          |

#### 2.4 GHz Co-Interference Note

The EMW3080B operates on 2.4 GHz � the same band as the Zigbee node link (IEEE 802.15.4, channels 11�26). Concurrent WiFi and Zigbee activity on the gateway can cause co-channel interference, degrading node PDR. Mitigations:

- **Channel separation:** Configure Zigbee on channels 25 or 26 (lowest WiFi overlap) and WiFi on channel 1, 6, or 11.
- **Staggered TX windows:** Gateway transmits WiFi upstream only after the Zigbee RX window closes each cycle.
- If co-interference proves problematic in dense deployments, switching to Ethernet (Option B) eliminates the RF conflict entirely.

## Option B � Ethernet Module (Alternative)

Wired Ethernet eliminates the 2.4 GHz co-interference issue and is the more reliable long-term option for a permanently installed gateway. The STM32U585 does not include an on-chip Ethernet MAC � all candidates below use SPI and include their own MAC+TCP/IP controller.

> **PHY-only ICs (LAN8720A, DP83848, KSZ8081) are incompatible** � they require a host-side Ethernet MAC, which the STM32U585 does not have.

### W5500 � Recommended Ethernet Module

The **WIZnet W5500** is the recommended choice if Ethernet is selected. It is the de facto standard SPI Ethernet controller for STM32 designs without an on-chip MAC.

**Key advantages:**

| Factor | Rationale |
|---|---|
| Hardware TCP/IP � 8 sockets | No Ethernet/ARP/IP/TCP logic on STM32; socket open ? connect ? send ? close maps directly to net_api.h |
| WIZnet ioLibrary | Official STM32 HAL driver � same integration level as EMW3080B |
| SPI at 80 MHz | 3� faster than ENC28J60 at 20 MHz; negligible for small JSON payloads |
| PoE path | W5500-PoE breakout boards add 802.3af/at PD: single Ethernet cable = data + gateway power |
| Compact hobbyist module | Readily available with RJ45 + magnetics; fits on a breadboard for dev kit prototyping |

**Connection to host MCU (custom PCB):**

`
Host MCU SPI    W5500 Breakout
--------------  --------------
SPI_SCK   ---?  SCLK
SPI_MOSI  ---?  MOSI
SPI_MISO  ?---  MISO
GPIO (CS) ---?  SCSn
GPIO      ---?  RSTn   (hardware reset)
GPIO (IN) ?---  INTn   (interrupt � packet received)
3.3V      ---?  3.3V

`

The SPI bus is dedicated to the network module � not shared with other peripherals. Enforced in the BSP peripheral ownership table.

## Selection

### Phase 1 � Development (B-U585I-IOT02A): EMW3080B

Selected because it is pre-installed on the dev kit with ST official driver support. No additional hardware required to begin gateway firmware development.

### Phase 2 � Custom PCB: Design for Both

The custom PCB (doc 3.7) shall support **both** EMW3080B (WiFi) and W5500 (Ethernet) via the same SPI bus with separate chip-select lines. Transport is selected at compile time via -DNET_TRANSPORT_WIFI / -DNET_TRANSPORT_ETHERNET (see doc 5.20). This allows field selection of WiFi or Ethernet without PCB respins.

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| 2.4 GHz co-interference (Zigbee + WiFi) | Medium | Medium | Channel planning (Zigbee ch 25/26, WiFi ch 1/6/11); or switch to Ethernet |
| EMW3080B firmware update breaks compatibility | Low | Low | Pin STM32CubeU5 submodule version; regression test before updating |
| WiFi credential provisioning in field | Medium | Medium | Store credentials in Secure flash partition; define provisioning flow |
| W5500 SPI contention with other peripherals | Low | Low | BSP peripheral ownership table enforces exclusive SPI access per module |
